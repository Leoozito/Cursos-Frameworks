Todos os frameworks mais recentes, giram em torno da ideia de remover o trabalho transacional minucioso, feito com o javascript do passado. Há pouco código javascript puro e sem efeito que você possa escrever e os reutilizar em outros projetos que não dependam de HTML e navegadores.
Muitos frameworks rastreiam a estrutura do frontend dentro do javascript e isso é útil para coisas como correção de erros.

O OWL permite escrever código mais simples, elegante e legível, mais facil de testar, de manter. o Odoo é conhecido como um dos ERPs mais fáceis do mundo.
O Odoo Web Library (OWL) é uma estrutura de interface do usuário pequena, destinada a ser a base para o Odoo Web Client.
O Odoo mudou sua estrutura para OWL, porque o OWL conclui com as necessidades do Odoo, não faz perder tempo com tempo de resposta.

Uma visão sobre o OWL

Principais características do OWL são:

  • Sistema de componentes declarativos.
  • Ganchos.
  • Mecanismo de reatividade granulado fino como Vue
  • Renderização assíncrona.
  • Fragmentos.

O Owl usa components como blocos de construção, os components são classes e funções JavaScript, para descrever como a interface do usuário deve aparecer na tela
OWL usa o mecanismo de template do Odoo ”Qweb”, para lidar com fragmentos e páginas HTML, de front-end, e com esse mecanismo fica mais dinamico.
Existe uma classe "Observer", que sempre observa as mudanças que acontecem em um objeto ou array.
O framework Owl usa Hooks para interagir dinamicamente, entre templates e components.
O gancho useState renderiza a visualização, toda vez que houver uma mudança de estado
