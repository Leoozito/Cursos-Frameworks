Pastas do next

Pasta node_modules é integrada em projetos que envolva npm, onde nela tem todas as depedencias, tudo que a gente executa no repositorio chama algo que está dentro desta pasta

O package.json como se fosse a config. geral do projeto, lá tem o nome do projeto e as dependencias. Qualquer projeto que seja criado com npm vai ter package.json

A pasta .nexté meio que uma pasta oculta, nosso projeto é gerado nela, por meio de mais pastas dentro dela.

Pasta pages, todo arquivo que sai de pages ele cria uma nova página, uma nova rota,então todo visual do chrome sai por meio de pastas e arquivos de lá

Também é ensinado que é importante criar uma pasta components, para adicionar os componentes criado que poderão ser reutilizados (ou seja ser usado em mais de uma page), e manter essa pasta separada da pasta "page"

Na pasta styles pode se criar um arquivo css

Pasta public onde se coloca imagens, icones.

Comandos

  • npx create-next-app <nome do projeto> : para criar a base do projeto
  • npm run dev : roda o projeto inteiro
  • npm run build : se deleta a pasta next. para não ser gerada com o build, e se tiver algum erro com o build , ele ira ser parado, para não ter o risco de colocar algo em produção que não está correto, essa função mostra detalhes sobre nossas pages, quais que são estaticos , quais que são dinamicos, conseguimos ver quantas páginas tem, o tamanho dos arquivos
  • npm run start : para ver o que será visto, ele roda o que ta dentro da pasta .next, faz com que mostre as paginas pré renderizada sem tempo de loagind (antes de rodar o start tem que buildar
  • Usando o código import Link (ou Image e outros{pesquisando o que vai com import}) from 'next/link' na aprte superior da para se criar um link para a pagina
  • Para caso quisermos adicionar mais de um elemento no export default, é só usar os parenteses e rfagmentos vazios ( < > < / > )
